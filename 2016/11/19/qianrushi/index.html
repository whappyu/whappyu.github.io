嵌入式相关知识笔记

发表于 2016-11-19   |   分类于 分享ing   |     |   次阅读

项目：智能停车终端系统
芯片：s5pc100
虚拟机：VMware
镜像：Ubuntu

嵌入式：以应用为中心，软硬件可裁剪的专用型计算机
知识点：c语言 linux操作系统 网络 数据库

（以下首字母大写的实际均为小写）

Linux使用：
1）快速打开终端
Ctrl+alt+t
2）放大终端
Shift + ctrl + +

3)缩小终端
Ctrl + -
4）命令提示符
用户名@主机名：当前目录 $

Linux:多用户
普通用户：非root用户 仅能操作自己的目录
超级用户：root用户 做任何操作
切换用户：su

Note：sudo 临时借用超级用户的权限

Note：linux永远只有一棵树
/:根目录
~：主目录
.：当前目录
..：上一级目录
../..:上上一级目录

Liunx系统命令：
关机：shutdown
例如：
Sudo shutdown -h now //立即关机
Sudo shutdown -h +10 //十分钟之后关机
重启：reboot
Sudo reboot
Sudo shutdown -r now //立即重启
查看当前日期：date
查看当日历：cal
查看当前工作目录：pwd
清屏：clear 或者ctrl+l

Liunx常用命令：
切换目录：cd
例如：
Cd / :切换到根目录
Cd ~：切换到主目录
Cd ：同上
Cd -：切换大最近一次工作的目录

显示目录的内容：ls
Ls：列出当前目录的所有内容
Ls -l：列出当前目录下的详细内容
Ls -a:列出当前文件内容（包含隐藏文件）
Note：以*开头的是隐藏文件

帮助手册：man手册

Man ls

退出：按q退出

创建文件：touch
例如：touch file1
Sudo Touch /file2

创建目录：mkdir
Mkdir dir1 dir2
Sudo mkdir /dir3

删除文件或者目录：rm
Rm file1 // 删除文件file1
Rm -r dir1 // 删除目录dir1
Rmdir dir1 // 删除目录dir1
Rm -rf * //强制性删除
Rm -f qwerdf //删除qwerdf，若文件不存在，不报错

修改权限：chmod
Ls -l：显示文件的详细信息

前面10个字母的含义：
第一个字母：表示文件的类型
-：普通文件
D：目录
C：字符设备
B：块设备
L：链接文件

2,3,4字符代表的是所属用户（当前创建此文件的用户）的权限
R:可读
W：可写
X:可执行
-：没有对应的权限

5,6,7字符代表当前所属用户的组的权限
8,9,10字符代表其他用户的权限

向文件中写入信息： > <重定向>
查看文件内容：cat
例如： cat file
以追加的方式：>>
写入自己的信息：echo

文件的复制：cp
Cp file1 file2 //将file1复制到file2
Cp -r dir1 dir2
文件的移动：mv
Mv file1 file2
Mv file3 file888 //假如没有file888文件 ，那就相当于给file3改名（重命名）

在dir1中创建file1和file2

向file1写入 hello farsight！，追加写入当前日历

复制dir1中的file1带根目录下的dir2

将dir1中的file1重命名为file3

删除dir2中file1

Vim编译器

Vim 3.c

命令行模式：
J 向下 k 向上 h 向左 l 向右
Yy：复制一行 10yy 复制10行
P： 粘贴
Dd：删除一行 10dd 删除10行
U：撤销

插入模式： I i A a O o
底行模式：
Esc ======> shift+;(:)
:w 保存
：q 退出
：wq 保存并退出 ：x
：q！ 强制性退出

C程序的编译过程：
预处理：替换所有以#号开头的文件
Gcc -E 4.c -o 4.i
编译：将预处理后的文件变成汇编文件
Gcc -S 4.i -o 4.s
汇编：将汇编文件生成了目标文件
Gcc -c 4.s -o 4.o
链接： 将目标文件与库文件进行链接
Gcc 4.o -o 4
编译：
Gcc 源文件 -o 可执行文件
运行：./可执行文件

在命令行模式下输入：gg=G 实现代码的缩进，使之美观

Char （1）

结构体： struct

开发板：
开发模式：通过网线将数据直接加载到内存中，进行测试
产品模式：通过usb烧录到成品
传输介质：串口线：端口监控

USB：烧录

带os嵌入式产品：
引导程序：uboot（vboot，supervivi）

内核：linux（winCE/mbed/vxworks）
根文件系统：busybox

配置网络：
虚拟机：图像化
配置文件：
Ifconfig查看一下 192.168.1.3

配置开发板网络：
在uboot交互界面输入：
Setenv ipaddr 192.168.1.2
Setenv netmask 255.255.255.0
Setenv serverip 192.168.1.3
Saveenv
通过print查看
通过ping 192.168.1.3
若显示** is alive 说明链接成功
若显示** is not alive 不成功
解决方法：
1）一定要保证虚拟机在桥接模式
2）关闭电脑wifi
3）关闭电脑防火墙
4）去查看电脑的无线网络，适当禁用

通过tftp下载内核：
1）将zImage 复制到 /tftpboot
2）在交互界面输入以下命令：
（下载并运行内核）
Setenv bootcmd tftp 20008000 zImage\;go 20008000
Saveenv
通过nfs挂载文件系统
1）将rootfs.tgz 复制到 /nfsboot
2）解压
Sudo tar xvf rootfs.tgz
3)加权限
Sudo chmod 777 /nfsboot -R
4)在交互界面输入：
Setenv bootargs root=nfs nfsroot=192.168.1.3:/nfsboot/rootfs ip= 192.168.1.2 init=/linuxrc console=ttySAC0,115200
Saveenv
